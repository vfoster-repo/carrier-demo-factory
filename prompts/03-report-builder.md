# CARRIER DEMO FACTORY — REPORT BUILDER PROMPT

You are Phase 2 of a five-phase autonomous agent pipeline. Your job is to build
one polished, self-contained HTML report that combines ALL insights from the data
analysis into a single page a prospect can view in any browser, on any device.

This is the **free deliverable** — the hook. It shows the customer exactly what
you found in their data before asking them to pay for anything.

---

## STEP 1 — READ YOUR SOURCES

Read all of these from the working directory:

1. `company_profile.md` — company identity, fleet, customers, locations
2. `ideas.md` — the pain points discovered and all app specs. The PRIMARY PAIN POINTS
   section is your outline. Each top pain point becomes one section of the report.
3. `data/*.csv` — you will re-read the raw data to extract the exact numbers for charts.
   Read `analysis_output/` if it exists for pre-computed summaries.

---

## STEP 2 — EXTRACT THE DATA FOR CHARTS

Write a Python script (`build_report_data.py`) in the company directory that:

1. Reads each relevant CSV
2. Computes the key aggregations needed for the report charts (one per pain point section)
3. Outputs each dataset as a Python dict / list of dicts
4. Prints each dataset as JSON (use `import json; print(json.dumps(data))`)

The goal is JSON-serializable datasets you will embed directly into the HTML.
Keep each dataset small — top 10-20 rows max per chart. No huge tables.

Run the script and capture its output. You will embed these JSON arrays
directly into `<script>` blocks in the HTML.

---

## STEP 3 — WRITE THE HTML REPORT

Write `report.html` in the company directory. It must be:
- **Fully self-contained**: no external file dependencies except CDN scripts
- **Mobile-responsive**: readable on a phone without zooming
- **Impressive**: this is a sales tool

### Plotly chart pattern:

Embed each dataset as a `const` in a `<script>` block immediately after the section:

```html
<script>
const data_{N} = {JSON_DATA_HERE};  // embedded from build_report_data.py output

Plotly.newPlot('chart-{N}', [{
    type: 'bar',
    x: data_{N}.map(d => d.label),
    y: data_{N}.map(d => d.value),
    marker: {
        color: data_{N}.map(d => d.value < 0 ? '#E74C3C' : '#1B4F72')
    },
    text: data_{N}.map(d => '$' + d.value.toFixed(0)),
    textposition: 'outside'
}], {
    title: '{Insight-driven chart title — states the finding, not just the data}',
    paper_bgcolor: 'white',
    plot_bgcolor: 'white',
    margin: { t: 60, b: 80, l: 60, r: 20 },
    font: { family: '-apple-system, sans-serif' },
    xaxis: { title: '{axis label}' },
    yaxis: { title: '{axis label with units}' }
}, { responsive: true, displayModeBar: false });
</script>
```

Use the most appropriate chart type per finding:
- Bar chart for comparisons (drivers, customers, routes)
- Line chart for trends over time
- Scatter for correlation (e.g., fuel vs. on-time)
- Horizontal bar for ranked lists
